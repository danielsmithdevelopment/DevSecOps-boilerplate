# k6

> Load testing with Prometheus remote-write to Mimir.

## In this repo

| Item | Value |
|------|-------|
| Image | `grafana/k6:0.57.0` |
| Script | `docker/observability/configs/k6/loadtest.js` |
| Compose service | `k6` (profile: `loadtest`) |

k6 does not run by default — activate with `--profile loadtest`.

## Quick start

From task-runner (includes observability stack):

```bash
cd task-runner
docker compose --profile loadtest run --rm k6
```

From observability stack directly:

```bash
cd docker/observability
K6_TARGET_URL=http://host.docker.internal:8080 docker compose --profile loadtest run --rm k6
```

## Configuration

| Variable | Description | Default |
|----------|-------------|---------|
| `K6_TARGET_URL` | URL to load test | `http://task-runner-1:8080` |
| `K6_OUT` | Output mode | `experimental-prometheus-rw` |
| `K6_PROMETHEUS_RW_SERVER_URL` | Mimir push endpoint | `http://mimir:9009/api/v1/push` |

### Script

`loadtest.js` — basic HTTP load scenario. Edit thresholds and VUs as needed.

## Making changes

1. Edit `configs/k6/loadtest.js` for scenarios, thresholds, or checks.
2. Set `K6_TARGET_URL` to [worker-cluster](../applications/worker-cluster.md) or other targets.
3. View results in Grafana → Mimir datasource (`k6_*` metrics).

## Integration

- Metrics remote-written to [Mimir](mimir.md)
- Visualize in [Grafana](grafana.md)
- Verified by [E2E script](../scripts/e2e-observability.md)
