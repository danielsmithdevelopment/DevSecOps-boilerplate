# k6

> Load testing and synthetic monitoring with Prometheus remote-write to Mimir.

## In this repo

| Item | Value |
|------|-------|
| Image | `grafana/k6:0.57.0` |
| Load script | `docker/observability/configs/k6/loadtest.js` |
| Synthetic script | `docker/observability/configs/k6/synthetic-check.js` |
| Compose services | `k6` (profile `loadtest`), `k6-synthetic` (profile `synthetic`) |

k6 does not run by default — activate with a Compose profile.

## Quick start

Load test (from task-runner):

```bash
cd task-runner
docker compose --profile loadtest run --rm k6
```

Synthetic check (low-rate production monitor):

```bash
cd docker/observability
docker compose --profile synthetic run --rm k6-synthetic
```

## Configuration

| Variable | Description | Default |
|----------|-------------|---------|
| `K6_TARGET_URL` | URL to check / load test | `http://task-runner-1:8080` |
| `K6_OUT` | Output mode | `experimental-prometheus-rw` |
| `K6_PROMETHEUS_RW_SERVER_URL` | Mimir push endpoint | `http://mimir:9009/api/v1/push` |

## Making changes

1. Edit `configs/k6/loadtest.js` for CI-style load scenarios.
2. Edit `configs/k6/synthetic-check.js` for critical-path journeys (login, checkout, …).
3. View results in Grafana → Mimir datasource (`k6_*` metrics).

## Integration

- Metrics remote-written to [Mimir](mimir.md)
